## 1. 项目概述

### 1.1 项目背景

PentestAgent 是一个基于大语言模型（LLM）的渗透测试自动化辅助系统。系统通过智能调度安全工具、结合专业知识库，辅助安全研究人员进行高效的渗透测试工作。

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| **人工确认机制** | 每一步操作都需要人类专家确认，确保安全可控 |
| **实时进度显示** | 使用 Rich 库显示美观的进度条和执行状态 |
| **知识库驱动** | 集成 PayloadsAllTheThings 知识库，提供专业指导 |
| **工具即插即用** | 直接调用系统已安装的安全工具，无需重新封装 |
| **详细日志记录** | 完整记录所有操作，便于审计和复现 |

### 1.3 适用场景

- ✅ 已获得授权的渗透测试项目
- ✅ 安全培训和演练环境
- ✅ 漏洞研究和验证
- ✅ 安全工具学习和测试

### 1.4 法律声明

```
⚠️ 警告：本系统仅限合法授权场景使用
未经授权对他人系统进行渗透测试属于违法行为
使用者需遵守当地法律法规
开发者不承担任何滥用责任
```

## 2. 设计原则

### 2.1 安全第一

- **人工确认**：所有敏感操作必须经过人工确认
- **命令透明**：显示完整的执行命令，无隐藏操作
- **风险评估**：每个操作都标注风险等级
- **可逆操作**：优先选择可逆或无损的测试方法

### 2.2 可观测性

- **实时反馈**：每一步操作都有清晰的视觉反馈
- **进度可视化**：使用进度条显示长时间任务
- **结果结构化**：输出结果易于理解和分析

### 2.3 模块化设计

- **松耦合**：各模块独立，易于维护和测试
- **可扩展**：方便添加新工具和功能
- **可配置**：通过配置文件定制行为

### 2.4 复用成熟组件

- **LangChain**：成熟的 LLM Agent 框架
- **Rich**：强大的终端 UI 库
- **系统工具**：直接调用已有的安全工具

## 9. 安全考虑

### 9.1 安全设计原则

```
┌─────────────────────────────────────────────────────────────────┐
│                       安全设计原则                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 最小权限原则                                                 │
│     • Agent 仅请求必要的权限                                     │

│     • 默认拒绝高风险操作                                         │

│     • 所有命令完整展示给用户                                     │
│     • 不隐藏任何执行细节                                         │
│     • 完整的日志记录                                             │
│                                                                 │
│  3. 可控性原则                                                   │
│     • 用户可随时中断执行                                         │
│     • 每一步都可修改或跳过                                       │
│     • 支持回滚和恢复                                             │
│                                                                 │
│  4. 审计原则                                                     │
│     • 记录所有操作和时间戳                                       │
│     • 保存完整的执行历史                                         │
│     • 支持操作回溯                                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 9.2 风险等级定义

| **中** | 🟡 黄色 | 可能产生日志或轻微影响 | 目录枚举、漏洞扫描 |
| **高** | 🟠 橙色 | 可能影响服务或产生流量 | 密码爆破、漏洞验证 |
